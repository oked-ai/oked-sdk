How it works

Three steps between your agent
and a bad day

OKed sits outside your agent process — so it can't be bypassed, even by a misbehaving model.

01
🔍

Agent calls a tool

Your AI agent attempts a tool call — reading files, running code, making API requests, or anything in between.

02
🛡️

OKed intercepts & classifies

The call is intercepted before execution. OKed classifies its risk tier and decides whether approval is needed.

03
📱

You get a notification

A rich push notification lands on your phone via Telegram or web push. The agent pauses until you decide.

04
✅

Features

Everything you need to trust
your AI agents

Built for developers who want the power of autonomous agents without handing over the keys to the kingdom.

📲

Real-time push notifications

Approval requests hit your phone instantly via Telegram or web push — wherever you are, whatever you're doing.

🎯

Smart risk classification

Every action is scored across four risk tiers — safe, warning, review, and high-stakes — so you only get pinged when it matters.

⚡

Works with your stack

Plug-and-play with Claude Code, Claude Agent SDK, and OpenClaw. Zero-config setup takes under two minutes.

🔒

Out-of-process security

OKed runs outside the agent process — it cannot be disabled or bypassed by the model, no matter what it tries.

📋

Full audit trail

Every decision is logged. Review what your agents attempted, what you approved, and what you blocked — all in one place.

🌐

Graceful degraded mode

If the approval server is unreachable, OKed falls back to a configurable default so your agents never hang silently.

Risk tiers

Four levels, one clear signal

OKed classifies every action before it runs. You define what needs your eyes and what can run automatically.

Safe

Auto-approved

Read-only operations. File reads, searches, API lookups — runs without interruption.

Warning

Heads up

Low-impact writes. Non-critical changes that run but are flagged in your audit log.

Review

Tap to confirm

Meaningful changes that pause for your approval before proceeding.

High Stakes

Hard stop

Destructive or irreversible actions. Always blocked until you explicitly approve.
